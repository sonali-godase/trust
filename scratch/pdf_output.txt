Module 2 Multi-Branch Audit & Accounting Architecture
This module enables the Trust to scale geographically while keeping financial records segregated 
and audit-ready for each location.
2.1. Decentralized Branch Management
•
Location-Specific Ledgers: Each branch (e.g., Branch A, Branch B) operates its 
own digital "cash book" and "bank book" within the system.
•
Branch-Level User Roles: Local managers can only view and input data for their 
specific branch, while the Central Trustee Dashboard retains a "Global View."
2.2. Individual Audit File Generation
•
Stand-alone Financials: Generate a separate Trial Balance, Income & 
Expenditure Account, and Balance Sheet for each branch.
•
Automated Annexures: Create branch-wise schedules for donations, expenses, and 
staff salaries, making it easy for the local auditor to verify records.
2.3. Centralized Compliance Sync
•
Consolidated Schedule IX & IX-C: While each branch has its own file, the system 
automatically aggregates all data into a single master report required for the Charity 
Commissioner’s annual filing.
•
Audit Lock System: Once a branch’s audit is finalized, the records for that year are 
"Locked" to prevent any backdated entries or tampering.

Module 9.Technical Specification & Security Sheet
This document outlines the infrastructure and security protocols designed to protect the Trust’s 
digital assets and devotee information.
9.1. Core Infrastructure & Reliability
•
Data Localization: In compliance with Indian data sovereignty preferences, all data is 
hosted on Tier-4 Data Centers located within India (Mumbai/Pune region).
•
High-Speed CDN: Implementation of a Content Delivery Network to ensure the website 
loads instantly across all branch locations, even on slower mobile networks.
9.2. Security & Data Protection
•
End-to-End Encryption: All data transmitted between the devotee and the server is 
protected by 256-bit SSL (Secure Socket Layer) Encryption.
•
Database Isolation: Branch-wise audit files are logically separated within the database to 
ensure that "Branch A" cannot access "Branch B" records without authorization.
•
Automated Backups: Daily encrypted "Point-in-Time" backups. In the event of any 
accidental deletion, the Trust's records can be restored to the exact state they were in 24 
hours prior.
9.3. Fraud Prevention & Access Control
•
Multi-Factor Authentication (MFA): Access to the Trustee Dashboard and Financial 
Records requires a password plus a one-time code (OTP) sent to the Trustee’s registered 
mobile number.
